# Stage 10B Arena Scale, Room Variety And Transitions

# Chrono Echo: Fractured Realms

## 1. Purpose

针对试玩反馈的四个问题：

1. 战斗房场景单一，玩起来枯燥。
2. 每关形式雷同，产生审美疲劳。
3. 战斗可活动范围太小。
4. 场景切换生硬。

Stage 10B 扩大战场、引入障碍物布局与敌人编成变化、加入统一的淡入淡出场景过渡。

## 2. Scope

本阶段实现：

- 战场扩大：
  - 战斗区域从 780x560 扩大到 1080x580（面积 +43%），HUD 改为浮在战场上方的半透明面板。
- 房间布局系统（确定性随机）：
  - 布局种子 = run seed + 节点 id 哈希：同一局内重进同一房间（含断点续玩）布局不变，不同局之间同一节点布局不同。
  - 四种障碍布局：Open（无）、Pillars（四柱）、Corridor（双横墙）、Scatter（散石）。
  - 障碍为静态物理体：阻挡玩家和敌人移动，双方弹道命中即毁。
  - 障碍贴图程序生成（灰阶 + 主题色 tint），随时间线主题变色。
  - Boss 房保持无障碍，保证弹幕可读。
- 敌人编成变化（同一种子驱动）：
  - 按深度从编成池随机抽取：浅层 2-3 敌，中层引入 Anomaly 组合，深层 4 敌混编。
  - 出生点随机分布在战场上半区，与玩家、彼此、障碍保持最小间距。
  - 精英房在编成外追加一只 Anomaly。
- 场景过渡：
  - 新增 `scene-transitions.ts`：`fadeInScene`（进场 260ms 淡入）与 `transitionTo`（200ms 淡出后切换），防抖避免重复触发。
  - 应用于主菜单、设置、记忆之树、时间树、战斗、事件、奖励、结算之间的全部主流程切换。

## 3. Design Rules

- 布局与编成必须确定性可复现，避免刷新改命。
- 障碍不得出现在玩家出生区和 Boss 房。
- 加大战场后敌人基础数值不变，深层编成数量增加即是难度曲线的一部分。
- 过渡时长控制在 200-260ms，不允许玩家感到拖慢。

## 4. Scope Control

本阶段暂不处理：

- 敌人绕障碍寻路（追击者贴障碍属可接受行为，反而形成掩体战术）。
- 多房间关卡与房间内波次。
- 外部美术资源（另行讨论）。
- 过渡动画的主题差异化。

- Playwright 验证：障碍物理体存在，玩家弹命中障碍被销毁。
- 截图核对：扩大后的战场、Pillars 与 Corridor 布局、主题 tint。
- 验证期间没有 browser page error。

- `npm run test:e2e` 11 项全部通过（含过渡动画时序）。
- Playwright 验证：
  - 同一节点重启两次：障碍位置与敌人编成完全一致，出生点误差在容差内（确定性成立）。
